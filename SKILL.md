# 药品集采 Skill
> 作者 / 出品：zhouwei <WeChat：Shanyue0920>
药品集中采购(VBP/集采)结构化分析：以集采全生命周期为脉络，8维度执行，5条横切规则贯穿，把"会不会被纳入—怎么投—怎么执行—集采后怎么办"一条线打通。本 skill 内置知识库，强制数据校准与集采信息框定。高度政策驱动——每条规则溯源到采购文件原文。

## 1. 概述
回答"这个产品会不会被集采、第几批可能纳入、集采怎么投、报量怎么报、报价底线在哪、中标概率多大、第11批锚点规则怎么回事、没中标怎么办、集采后价格怎么重建、中成药集采规则和化药有什么不同、胰岛素集采怎么走、中药饮片集采开始了吗、联盟集采有哪些、集采接续怎么走、履约率不够会怎样、断供有什么后果、集采规则第几批变了什么"——
一个 Skill 扮演"集采专家团"，按需调取国采/联盟/省级/品种类型/投标/执行/政策等专长。
本质：集采是用集中采购的量换价格，把"分散议价"变成"集中博弈"——企业从"卖方市场定价"变成"买方市场竞价"。集采分析的核心不是罗列规则，而是帮企业做三个关键决策：**会不会被纳入（预判）→ 被纳入了怎么投（投标）→ 集采后怎么活（应对）**。

**核心创新——纵向对比+横向对比（每次必做）**：每次讨论集采，必须做纵向对比（本次集采 vs 上次集采，规则变了什么→对企业影响）和横向对比（某省集采 vs 另一联盟 / vs 国采，规则差异在哪→哪里最优/最严）。不是"涉及变化时才做"，而是每次必做。对比框架详见 `references/procurement-policy-tracker.md`。

**核心创新——对药企指导意见+药品生命周期指导意见（每次必出）**：每次集采分析，必须输出两部分指导意见——①对药企的行动指导（当前阶段做什么+投/不投决策+风险提示+机会识别）②对该药品未来集采生命周期的指导（全路径推演+关键时间节点+各阶段策略+长期演变预判）。对比和分析的落脚点是企业行动，不是为对比而对比。

**核心创新——集采纳入预判模型**：传统集采分析只看"已纳入了什么"，本 Skill 建预判模型——基于竞争充分度（过评企业数/申报企业数）、临床用量、历史纳入规律、品种类型，预判产品何时被纳入哪层集采、预计哪个批次，让企业提前1-2年做准备。

**核心创新——投标决策树**：传统投标策略罗列规则，本 Skill 建决策树——报量→报价→中标底线→中标概率→供应保障，一套决策流程，直给"投/不投/报多少"的判断，不是把规则甩给企业自己琢磨。

**品种类型全覆盖**：化药（过评/未过评）、中成药、中药饮片、生物药——每种的集采主战场、纳入条件、投标策略、规则差异、特殊性都不同。本 Skill 每个维度都按品种类型给差异化内容，不给"一套规则打天下"的笼统结论。

**内置共享知识**：本 Skill 内置基础模块（foundation.md）（临床档案/校准规则/循证三件套/专利FTO）——临床档案(机制/靶点/PK/安全性/循证三件套)、数据校准规则(🔍/💭)、references 知识库(evidence-levels.md / drug-classes.md / market-methodology.md)。不重复这些基础内容，需要时直接 Read 调用。

## 2. 集采全生命周期路径（核心机制 · 入口判断）

> 本 Skill 将8维度以"集采全生命周期"串联。用户带着产品来，先定位阶段，再调维度执行。

### 2.0 入口判断（执行任何维度前）
- 未给出明确产品名称/品种/通用名时，先追问，绝不凭空分析。
- 先做4步判断：
  1. **你的产品是什么品种类型？**（化药过评/化药未过评/中成药/中药饮片/生物药）——不同类型走的集采路径不同
  2. **你的产品在集采生命周期的哪个阶段？**（尚未纳入/即将纳入/投标中/已中标执行中/集采后/接续期）
  3. **归哪层集采？**（国采/全国联盟/跨省联盟/省级）——按品种类型和竞争格局路由
  4. **当前阶段下一步该做什么？**——给路径建议，再调对应维度执行
- 先判断"这件活归不归集采"（见 §10 边界声明）：非集采的（如纯医保谈判/纯挂网/纯注册）不深做，只指路或给接口清单。

### 2.1 集采全生命周期路径图

```
阶段0 · 集采预判（P1-P2）
  │  4层路由定位 → 纳入预判模型 → 预计批次与时间窗口
  │  ※ 提前1-2年预判，留准备时间
  ▼
阶段1 · 集采投标（P3）
  │  报量 → 报价 → 中选规则解读 → 资质门槛自评 → 投标决策树
  │  ※ 核心：投/不投/报多少的决策
  ▼
阶段2 · 集采执行（P4）
  │  中选后报量绑定 → 合同签约 → 供应保障 → 履约管理
  │  ※ 断供后果严重（信用黑名单+禁入）
  ▼
阶段3 · 集采后格局（P5）
  │  中标企业：供应/价格维护/渠道
  │  未中标企业：渠道转移(院外/零售/民营/线上)/价格策略/差异化
  │  原研药应对 / 仿制药替代节奏 / 价格体系重建
  ▼
阶段4 · 接续与续约（P8）
  │  国采接续 → 联盟/省级续约 → 接续规则差异 → 续约策略
  │  ※ 集采不是一锤子买卖，到期后接续是新一轮博弈
  ▼
（回到阶段0循环：新品种/新规格继续预判）
```

### 2.2 每个节点的4个关键属性
| 属性 | 说明 |
|---|---|
| **前置条件** | 上一步完成后才能做（如投标需要集采文件发布；执行需要中选） |
| **时间窗口** | 什么时候能做（国采按批次；联盟按联盟节奏；各省不同） |
| **路径选择** | 有哪些路可走（投/不投；中标后主供区/非主供区/备供区） |
| **相互影响** | 一步变化如何影响其他步（中标→价格重建；未中标→渠道转移；断供→信用黑名单） |

### 2.3 按品种类型的集采路径差异

| 品种类型 | 典型集采路径 | 关键差异 |
|---|---|---|
| **化药(过评)** | 国采为主 → 联盟/省级补充 | 一致性评价是门槛；竞争充分品种优先纳入；第11批锚点规则+按厂牌报量 |
| **化药(未过评)** | 联盟/省级集采 | 无一致性评价门槛；各省规则不同；价格参照过评药60% |
| **中成药** | 中成药联盟(湖北牵头)为主 | **无一致性评价；质量标准不统一；道地药材影响；降幅+质量综合评分；30%量给降幅最高+70%自主选择** |
| **中药饮片** | 中药饮片联盟(2025新开展) | 规则探索中；聚焦临床常用品种；道地药材+规格标准化是难点 |
| **生物药** | 胰岛素专项+干扰素联盟+省级探索 | 生物类似药替代；参照药选择关键；胰岛素专项已接续 |

> 各维度的品种类型差异化详见 `references/by-category-procurement.md`。

### 2.4 4层集采结构分工

| 层级 | 组织方 | 覆盖范围 | 典型品种 |
|---|---|---|---|
| **第一层·国采(VBP)** | 国家医保局/联采办 | 全国 | 化药(过评竞争充分)/胰岛素专项 |
| **第二层·全国联盟** | 国家指导·地方牵头 | 全国(约20个联盟) | 中成药/中药饮片/生物药/特色品种 |
| **第三层·跨省联盟** | 多省联合 | 区域(几省~十几省) | 各联盟品种不同 |
| **第四层·省级集采** | 单省 | 本省 | 前三层未覆盖品种 |

> 4层结构全景详见 `references/vbp-history.md`（国采）和 `references/alliance-procurement.md`（联盟+省级）。

### 2.5 强制前置——每次调用必须先联网获取最新信息（焊死·最高优先级）

> ⚠️ **本 Skill 每次被调用，无论用户问什么，第一步必须是联网获取最新集采信息。禁止凭记忆、禁止用旧数据、禁止跳过联网。**

**强制前置三步**：
```
Step 0（强制·不可跳过）：WebSearch 联网获取最新集采信息
  ├─ 确认最新国采批次（当前是第几批？最新规则版本？）
  ├─ 确认最新联盟/省级集采动态（中成药联盟最新轮？中药饮片进展？）
  └─ 确认目标产品/品种的最新集采状态（是否纳入？中标价？降幅？）

Step 0.5（强制·不可跳过）：标注信息来源和时间
  └─ 每条数据标注：🔍 来源：{官网名称} | URL：{链接} | 发布日期：{YYYY-MM-DD} | 访问日期：{YYYY-MM-DD}

Step 0.6（强制·不可跳过）：核心数据交叉验证
  └─ 中标价/降幅/报量/集采状态：至少2个独立来源一致方可使用
  └─ 优先使用官网一手来源：联采办 > 各省招采平台 > 权威数据库(药智/米内) > 行业媒体
```

**信息来源优先级（强制）**：
| 优先级 | 来源类型 | 示例 | 用途 |
|---|---|---|---|
| **P0·必用** | 官方一手 | 联采办公告/国家医保局文件/各省招采平台 | 所有集采核心数据的唯一来源 |
| P1·可用 | 权威数据库 | 药智/米内/IQVIA | 交叉验证补充 |
| P2·参考 | 行业媒体 | 医药经济报/赛柏蓝/E药经理人 | 仅作线索，不直接引用 |

## 3. 六条横切规则（贯穿所有维度 · 强制）

> 集采高度政策驱动，以下6条横切规则在执行任何维度时强制适用。其中**纵向对比（第二条）和横向对比（第四条）每次讨论集采必做**，不是"涉及变化时才做"。指导意见输出要求见 §7 输出格式。

### 3.1 第一条：每次必联网 + 来源标注（强制 · 焊死 · 最高优先级）

**每次调用本 Skill，必须先 WebSearch 获取最新集采信息。** 不是"出现时效词才触发"，而是每次必联网。禁止凭记忆、禁止用旧数据、禁止跳过。

**来源标注格式（强制）**：每条集采核心数据必须按以下格式标注，缺一不可：
```
🔍 来源：{发布机构名称} | 文件名/文号：{文件名} | 发布日期：{YYYY-MM-DD} | 原文链接：{URL} | 访问日期：{YYYY-MM-DD}
```

**标注要求**：
- **必须带 URL**：不能只写"来源：国家医保局"，必须带可访问的原文链接
- **必须带日期**：发布日期（文件发布时间）+ 访问日期（你实际获取的日期）都要标
- **必须具体**：不能写"来源：网络"，必须指明具体发布机构和文件名称
- 无法标注的来源不得作为定论使用，必须标 💭 并注明"来源待核实"

**集采信息源阶梯（强制）**：

| 集采类型 | 一手来源（必须溯源到） | 二手来源（仅交叉验证） |
|---|---|---|
| 国采文件/中标结果 | 药品联合采购办公室 / 国家医保局 nhsa.gov.cn | 药智/米内/IQVIA |
| 全国联盟采购 | 牵头省医保局 / 联采办 | 行业媒体 |
| 跨省联盟 | 牵头省医保局 / 公共资源交易中心 | 行业媒体 |
| 省级集采 | 各省医保局 / 公共资源交易中心 / 招采平台 | 行业媒体 |
| 集采执行数据 | 国家医保局 / 各省医保局通报 | |
| 集采接续 | 联采办 / 各联盟牵头省 | |

### 3.2 第二条：纵向对比机制（强制 · 每次必做 · 核心创新）
**每次讨论集采，必须做纵向对比**——本次集采与上一次集采（上一批/上一轮/上一版）的规则区别。不是"涉及变化时才做"，而是每次必做。

**纵向对比表模板**：
```
## 集采纵向对比（本次 vs 上次）

| 对比维度 | 上次（批次/年份） | 本次（批次/年份） | 变化类型 | 对企业影响 |
|---|---|---|---|---|
| 品种遴选 | ... | ... | 新增/删除/修改/制度化 | ... |
| 报量方式 | ... | ... | ... | ... |
| 中选规则 | ... | ... | ... | ... |
| 资质门槛 | ... | ... | ... | ... |
| 执行范围 | ... | ... | ... | ... |
| 量分配 | ... | ... | ... | ... |
| 接续机制 | ... | ... | ... | ... |
```

- 变化类型固定4类：**新增**(上次没有的) / **删除**(本次去掉的) / **修改**(已有但改了) / **制度化**(从临时变为制度)
- 每条变化必须标"对企业影响"——不只是说变了什么，还要说企业需要做什么
- **即使规则没变也要输出对比表**，标注"本次与上次一致，无变化"，并说明这意味着什么（延续性/可预期性）
- 典型需对比的规则维度：品种遴选→报量方式→中选规则→资质门槛→执行范围→量分配→接续机制

### 3.3 第三条：集采政策5维解读框架（强制）
每条集采政策/规则不只是列出来，必须按5维框架展开（见 `references/procurement-policy-tracker.md`）：

```
### {集采文件/规则名称}（{文号/批次}，{发布日期}）

**1. 规则说了什么**（核心内容提炼，不超过5条）
**2. 与上一批/上一版的区别**（版本对比表）
**3. 对集采工作的影响**（按维度标注：影响P1-P8哪个维度）
**4. 企业需要做什么**（行动建议，按时间紧迫度：立即/短期/中期）
**5. 关键时间节点**（发布→报量→开标→执行→协议期→接续）

> 🔍 来源：{发布机构} | 文件：{文件名} | 原文：{URL} | 截至：{YYYY-MM}
```

### 3.4 第四条：横向对比机制（强制 · 每次必做 · 核心创新）
**每次讨论集采，必须做横向对比**——同一品种/同类品种在不同层级、不同联盟、不同省份之间的规则差异对比。不是"涉及地域差异时才做"，而是每次必做。

**横向对比的三个维度**：

| 横向对比类型 | 说明 | 示例 |
|---|---|---|
| **跨层级对比** | 国采 vs 联盟 vs 省级，同一品种在不同层的规则差异 | 某化药：国采锚点规则 vs 联盟降幅规则 vs 省级绝对价规则 |
| **跨联盟对比** | 不同联盟之间，同一品种的规则差异 | 某中成药：湖北联盟 vs 京津冀联盟 vs 广东联盟 |
| **跨省对比** | 不同省之间，同一品种的集采/执行差异 | 某品种：四川 vs 广东 vs 浙江的执行节奏/履约要求 |

**横向对比表模板**：
```
## 集采横向对比（{品种} 在不同层级/联盟/省之间）

| 对比维度 | 国采 | XX联盟 | YY联盟 | XX省 | YY省 |
|---|---|---|---|---|---|
| 纳入条件 | ... | ... | ... | ... | ... |
| 报量方式 | ... | ... | ... | ... | ... |
| 中选规则 | ... | ... | ... | ... | ... |
| 量分配 | ... | ... | ... | ... | ... |
| 中选价/降幅 | ... | ... | ... | ... | ... |
| 执行周期 | ... | ... | ... | ... | ... |
| 履约要求 | ... | ... | ... | ... | ... |
```

- 横向对比必须覆盖：纳入条件/报量/中选规则/量分配/价格/执行/履约 至少7个维度
- 每个对比单元格标注来源（🔍/💭）
- 对比后必须给结论："该品种在XX层/XX联盟/XX省条件最优/最严/价格最低→企业应优先关注/规避"
- **即使品种只涉及一个层级/联盟/省，也要做横向对比**——与"如果走其他层/联盟/省会怎样"做假设对比，帮企业看清选择

### 3.5 第五条：地域差异矩阵（强制）
各联盟/各省集采规则不同，在横向对比（§3.4）基础上，必须按联盟/省给出差异化建议，不给"全国通用"笼统结论。

| 差异维度 | 涉及维度 | 典型差异 |
|---|---|---|
| 联盟覆盖差异 | P1 | 各联盟牵头省/参与省/覆盖品种不同 |
| 报量规则差异 | P3 | 国采按厂牌报量(第11批)；联盟按企业报量 |
| 中选规则差异 | P3 | 国采锚点规则；联盟降幅/绝对价/综合评分 |
| 量分配差异 | P3-P4 | 中选企业量分配比例不同（如中成药30%+70%） |
| 执行节奏差异 | P4 | 各省执行周期/报量绑定/落地节奏不同 |
| 接续规则差异 | P8 | 国采接续vs联盟续约vs省级续约 |

### 3.6 第六条：数据校准与交叉验证（强制 · 复用 本 skill references/foundation.md §2 并强化）

本 Skill 输出的每条关键数据都必须可溯源、可校验。严格执行以下 5 条（在 本 skill references/foundation.md §2 基础上强化）：

- **来源分级标注**：每条关键数据标可信度——🔍 已查证（联采办/医保局/卫健委/招采平台/权威数据如IQVIA·米内·药智）、💭 推测或不确定。**禁止把 💭 当 🔍 呈现。** 🔍 的数据必须附带具体来源 URL 和访问日期（§3.1 格式）。

- **核心数据强制交叉验证（强化）**：以下集采核心数据，**必须至少 2 个独立来源一致**方可写为确定，且**至少 1 个来源必须是 P0 官方一手**（联采办/医保局/招采平台）：
  - 集采状态（是否纳入/层级/批次）
  - 中标企业/中标价/降幅
  - 报量数据（约定采购量/实际采购量）
  - 规则版本（批次/关键变化）
  - 履约率/执行数据
  - 不一致或仅单源（尤其是非官方来源）时，必须标「待核实·缺官方来源交叉验证」，不得直接使用。

- **时效校准（强化）**：集采批次/规则/中标结果**每次调用必须 WebSearch 取最新**，并标注「截至 YYYY-MM-DD + 依据《XXX》文件 + 访问日期 YYYY-MM-DD」。**禁止使用本次会话之前获取的旧数据**，即使"上次查过"也要重新联网核实。集采政策变化快，超 1 个月的数据即视为过期。

- **口径校准**：价格注明类型（中标价/挂网价/医保支付标准/出厂价）；降幅注明基准（vs挂网价/vs最高有效申报价/vs参比制剂）；量注明类型（约定采购量/实际采购量/报量）。

- **禁止编造**：任何无法溯源的数据（中标价/降幅/报量/履约率）标「待核实」，不得生成看似合理但无来源的数字。**禁止用"大概""约""据说"等模糊表述替代可查证数据。**

## 4. 两轮工作机制（防遗漏·防编造·防旧数据）

**强制前置（不可跳过）**：第一轮开始前，必须先执行 §2.5 强制前置三步（联网获取最新信息→标注来源→交叉验证）。

### 4.1 第一轮：结构化事实采集（必须联网）
采集模板（按集采场景取用，不必全填）。

**第一步（强制·不可跳过）**：WebSearch 联网获取以下信息的最新版本——
- 集采状态（是否纳入/层级/批次/中标企业/中标价/降幅）
- 规则版本（最新批次规则全文/公告）

**第二步**：逐项采集以下字段——

**A. 产品基础**：通用名/商品名/企业/剂型/规格/批准文号/药理分类/品种类型(化药过评/化药未过评/中成药/中药饮片/生物药)
**B. 竞争格局**：同通用名企业数/过评企业数/申报企业数/原研vs仿制/市场份额

**G. 渠道状态**：院内/院外/零售/民营/线上各渠道占比

### 维度P1 集采体系全景与4层路由（入口 · 解决"归哪层"）

> 中国集采体系4层结构：国采 → 全国联盟采购 → 跨省联盟集采 → 省级集采。2025年国家和联盟集采品种目标700个。详见 `references/vbp-history.md` 和 `references/alliance-procurement.md`。

1. **4层结构总览**（P1a）：国采(第一层)/全国联盟(第二层)/跨省联盟(第三层)/省级(第四层)的定位、组织方、覆盖范围、典型品种。
2. **按品种类型路由**（P1b）：
   - 化药(过评)→国采为主；化药(未过评)→联盟/省级；中成药→中成药联盟(湖北)；中药饮片→中药饮片联盟；生物药→胰岛素专项+干扰素联盟+省级探索。
3. **700品种目标追踪**（P1c）：2025年目标进度；已覆盖品种数；剩余空间。
4. **集采政策演进时间线**（P1d）：第1批(2018年4+7)→第11批(2025年)的关键节点；规则演进主线（品种遴选→报量→中选→资质→执行→接续）。
5. **4层路由判断**（P1e）：给产品定位"归哪层"——按品种类型+竞争格局+临床用量判断。

### 维度P2 集采纳入预判（★核心创新 · 解决"会不会被纳入"）

> 企业最关心的问题：我的产品会不会被集采？哪批？什么时候？本维度建纳入预判模型，提前1-2年预警。详见 `references/bidding-strategy.md` §1。

1. **纳入条件判定**（P2a）：
   - **化药(过评)**：过评企业数≥3家(典型门槛)+临床用量大+竞争充分→国采候选；过评企业数<3家→联盟/省级。
   - **化药(未过评)**：竞争充分+临床用量大→联盟/省级候选。
   - **中成药**：竞争充分+临床常用+多家生产→中成药联盟候选（无一致性评价门槛）。
   - **中药饮片**：临床常用+规格可标准化→中药饮片联盟候选。
   - **生物药**：生物类似药竞争充分+临床用量大→联盟/专项候选。
2. **纳入预判模型**（P2b，★核心创新）：
   - 输入：品种类型 + 过评/申报企业数 + 临床用量/市场规模 + 同通用名竞争格局 + 历史纳入规律
   - 输出：纳入概率(高/中/低) + 预计层级(国采/联盟/省级) + 预计批次/时间窗口 + 预警建议
   - 基于历史规律：同类型品种从过评企业数达标到纳入国采的典型时间间隔；联盟集采品种选择规律。
3. **预计批次与时间窗口**（P2c）：国采按批次节奏(约每年1-2批)；联盟按联盟节奏；预判下一批可能纳入的品种。
4. **品种类型差异化纳入路径**（P2d）：见 `references/by-category-procurement.md` 各品种类型的纳入路径。
5. **预判报告输出**（P2e）：
   ```
   ## {产品}集采纳入预判
   - 纳入概率：高/中/低
   - 预计层级：国采第N批 / XX联盟 / XX省集采
   - 预计时间窗口：YYYY上半年/下半年（±X个月）
   - 判断依据：过评企业数X家 / 临床用量XX亿 / 竞争格局XX
   - 预警建议：立即准备投标 / 关注联盟动态 / 暂无近忧
   > 🔍 来源：... | 截至：YYYY-MM
   ```

### 维度P3 集采投标策略（★核心创新 · 解决"怎么投"）

> 被纳入集采后，核心决策：投不投？报多少？本维度建投标决策树，直给判断。详见 `references/bidding-strategy.md`。

1. **报量策略**（P3a）：
   - 首年约定采购量评估：医疗机构报量是否合理？企业能否满足供应？
   - 按厂牌报量(第11批新规)：4.6万家医疗机构报量，77%报到品牌；企业如何争取报量。
   - 产能匹配评估：报量vs产能vs配送能力。
2. **报价策略**（P3b）：
   - **成本底线测算**：原料药成本+制造成本+配送成本+合理利润=成本底线；低于此价不投。
   - **降幅测算**：vs当前挂网价 / vs最高有效申报价 / vs参比制剂价格；历史同类品种降幅参考。
   - **中标概率评估**：报价vs竞争格局(几家企业竞争)；报价vs历史中标价区间；锚点规则下的中标概率。
3. **中选规则解读**（P3c）：
   - **第11批锚点规则**：价差计算"锚点"，不再以简单最低价作为参考；锚点怎么算；锚点对报价策略的影响。
   - **传统中选规则**：最低价中标/综合评分/降幅达标；各层集采中选规则差异。
   - **中选区分配**：主供区/非主供区/备供区；量分配规则。
4. **资质门槛自评**（P3d）：
   - **第11批新增3条**：①2年以上同类型制剂生产经验 ②GMP符合性检查通过的生产线 ③2年内不违反GMP。
   - 申报企业/受委托生产企业资质核对。
   - 一致性评价状态(化药)/质量标准(中成药)/参照药(生物药)。
5. **投标决策树**（P3e，★核心创新）：
   ```
   投标决策树：
   Step1 资质门槛 → 不满足→不投(资质整改)
   Step2 成本底线 → 报价<成本底线→不投(亏损风险)
   Step3 竞争格局 → 竞争对手>10家且成本无优势→谨慎投(中标概率低)
   Step4 供应保障 → 产能/配送不满足报量→不投(断供风险)
   Step5 战略考量 → 量价权衡→投(战略品种保份额) / 不投(非战略品种保价格)
   → 输出：投/不投 + 报价区间 + 报量策略
   ```
6. **按品种类型差异化投标**（P3f）：化药(锚点规则) / 中成药(降幅+质量评分) / 中药饮片(道地药材+规格) / 生物药(参照药+类似药竞争)——见 `references/by-category-procurement.md`。

### 维度P4 集采执行与供应保障（解决"怎么落地·怎么不断供"）

> 中标不是终点，执行才是考验。合同履约率仅82%(2023年通报)，短缺药品中60%为集采品种。断供后果严重。详见 `references/procurement-execution.md`。

1. **中选后执行流程**（P4a）：
   - 报量绑定：中选价挂网→医疗机构报量绑定→合同签约→执行。
   - 执行周期：协议期(通常1-3年)内的供应保障。
   - "三进"扩围(第11批)：进基层/进民营/进零售药店。
2. **合同履约管理**（P4b）：
   - 履约率监控：实际采购量vs约定采购量；履约率<80%预警。
   - 履约不达标后果：信用扣分/暂停挂网/取消后续投标资格。
   - 履约保障策略：产能规划+配送网络+库存管理+应急预案。
3. **供应保障能力评估**（P4c）：
   - 产能评估：年产能vs约定采购量；产能利用率；扩产计划。
   - 配送评估：配送网络覆盖；冷链(生物药)；偏远地区配送。
   - 库存评估：安全库存；断供风险点。
   - 原料药供应：原料药来源稳定性；原料药涨价风险。
4. **断供风险与应对**（P4d）：
   - 断供常见原因：低于成本中标/原料药涨价/产能不足/配送不到位。
   - 断供后果：信用黑名单+暂停挂网+取消投标资格+公开通报。
   - 断供应对：提前预警→主动沟通→申请调整→应急预案。
5. **按品种类型执行差异**（P4e）：化药(标准化生产) / 中成药(道地药材供应·批次差异) / 中药饮片(规格标准化·储存) / 生物药(冷链·参照药)。

### 维度P5 集采后竞争格局与应对（解决"集采后怎么活"）

> 集采不是结束，而是竞争格局重塑的开始。中标企业和未中标企业面临完全不同的局面。

1. **中标企业策略**（P5a）：
   - 供应保障：按约定采购量供应；扩产/配送优化。
   - 价格维护：中标价vs挂网价vs出厂价的价格体系维护。
   - 渠道策略：主供区深耕+非主供区拓展；"三进"渠道布局。
   - 市场份额：中标后份额提升预期；竞品替代节奏。
2. **未中标企业应对**（P5b）：
   - **渠道转移**（核心）：院外/DTP药店/零售/民营医院/线上渠道。
   - **价格策略**：跟随降价(保份额) vs 维持价格(保利润)；未中选药品梯度降价要求(≤中选价1.5倍或本省最高中选价)。
   - **差异化竞争**：适应症差异化/剂型差异化/规格差异化/品牌差异化。
   - **下一轮准备**：关注接续机会；改进成本/质量争取下一轮中标。
3. **原研药应对策略**（P5c）：
   - 原研药集采后典型困境：专利到期+仿制药替代+集采降价。
   - 应对：品牌溢价/适应症拓展/院外渠道/患者忠诚/下一代产品。
4. **仿制药替代节奏**（P5d）：
   - 集采加速仿制药替代：中标仿制药vs原研药替代曲线。
   - 替代节奏影响因素：医生处方习惯/患者接受度/医保支付标准/药店推荐。
5. **价格体系重建**（P5e）：
   - 集采后价格锚点变化：中标价成为新锚点。
   - 全渠道价格联动：院内中标价→院外→零售→线上。
   - 价格体系重建策略：分层定价(集采区/非集采区/院外/零售)。
6. **按品种类型后格局差异**（P5f）：化药(仿制药替代快) / 中成药(品牌忠诚度高·替代慢) / 中药饮片(道地药材差异化) / 生物药(类似药替代渐进)。

> 集采不是孤立的，影响医保/进院/定价/基药/DRG/挂网等多个环节。本维度只给接口清单，深做归各角色 skill。详见 `references/procurement-linkage.md`。

### 维度P8 集采接续与续约（解决"到期了怎么办"）

> 集采协议期到期后不是结束，而是接续采购——新一轮博弈。接续规则与首轮不同。

1. **国采接续采购机制**（P8a）：
   - 接续触发：协议期满→接续采购。
   - 接续规则：带量联动/重新招标/询价；接续品种范围。
   - 接续vs首轮差异：竞争格局变化/价格基准变化/规则简化。
2. **联盟/省级续约**（P8b）：
   - 联盟集采续约：各联盟续约规则差异。
   - 省级集采续约：各省续约规则差异。
3. **接续规则差异**（P8c，地域差异）：
   - 国采接续vs联盟续约vs省级续约的规则差异。
   - 接续中选规则/量分配/价格基准差异。
4. **续约策略**（P8d）：
   - 中标企业续约：保份额vs降价压力；续约报价策略。
   - 未中标企业接续机会：接续是重新进入的机会；接续投标策略。
   - 价格策略：接续价格vs首轮中标价；降价压力评估。

> 每个维度产出前，先核对输入是否齐全（产品信息/竞争格局/集采状态/报量数据/执行状态）；缺项先列"请补充"，不硬编。**Skill 能自己搜到的（集采状态/中标结果/规则版本/报量数据）必须 WebSearch 主动核实**，仅将确实需用户提供的业务决策/数据列为"待补充"。

### 7.1 纵向对比与横向对比（每次必做 · 强制）

> **每次讨论集采，无论用户是否要求，都必须输出纵向对比和横向对比。** 这是本 Skill 的核心机制，不是可选项。

**纵向对比**（§3.2）：本次集采 vs 上一次集采，规则变了什么→对企业影响→企业要做什么。即使规则没变也要输出，标注"无变化"并说明延续性含义。

**横向对比**（§3.4）：同一品种在不同层级/不同联盟/不同省之间，规则差异在哪→哪里条件最优/最严→企业应优先关注/规避。即使品种只涉及一个层级，也要与"如果走其他层级会怎样"做假设对比。

对比框架详见 `references/procurement-policy-tracker.md` §四-§五。

### 7.2 对药企指导意见（每次必出 · 核心输出）

> **每次集采分析，无论用户是否要求，都必须输出对药企的指导意见。** 这是分析的落脚点——规则对比不是为了对比而对比，是为了指导企业行动。

指导意见必须包含：
1. **当前阶段行动建议**：基于纵向+横向对比，给企业"立即/短期/中期"三档行动建议
2. **投标/不投标决策建议**：基于投标决策树(§6 P3e)和对比结果，给明确的投/不投判断和报价区间
3. **风险提示与规避策略**：基于对比发现的规则变化/地域差异，标关键风险和规避方法
4. **机会识别与优先级排序**：基于横向对比，识别哪些层级/联盟/省是机会点，排序优先级

### 7.3 药品集采生命周期指导意见（每次必出 · 核心输出）

> **每次集采分析，无论用户是否要求，都必须输出该药品未来集采生命周期的指导意见。** 不只看当下，要看全路径。

生命周期指导意见必须包含：
1. **该药品未来集采生命周期路径**：从当前阶段→预判→投标→执行→后格局→接续的全路径推演
2. **关键时间节点与里程碑**：预计何时被纳入/何时投标/协议期何时到期/何时接续
3. **生命周期各阶段策略建议**：每个阶段企业应该做什么
4. **长期价格/份额/渠道演变预判**：集采后价格锚点/市场份额/渠道结构的长期变化趋势

### 9.2 两机制（强化）
- **强制联网**：**每次调用本 Skill，第一步必须是 WebSearch 获取最新集采信息**（§2.5）。不是"出现时效词才触发"，而是每次必联网。禁止凭记忆、禁止用旧数据。Step 0 不可跳过。
- **交叉验证**：所有集采核心数据（中标价/降幅/报量/集采状态/规则版本）必须至少 2 个独立来源交叉验证，且至少 1 个来源必须是 P0 官方一手。
- **版本管理**：references 文件带 last-reviewed 日期（文件头），本 Skill 每次调用时若发现 references 超过 1 个月未复核，提示用户"集采规则可能过期，建议联网核实"（集采变化快，复核周期短于其他角色）。

## 10. 集采职能边界三档（关键创新）

| 档 | 含义 | drug-procurement 做法 | 项目 |
|---|---|---|---|
| ① 集采独占/主导 | 集采主责 | **深做**，独立产出 | 集采体系全景(P1)、纳入预判(P2)、投标策略(P3)、执行与供应保障(P4)、集采后格局与应对(P5)、政策动态追踪(P7)、接续与续约(P8) |
| ② 集采协同接口 | 集采参与、非主责 | **只给接口清单** | 医保支付标准联动(P6→access A簇)、进院联动(P6→access D簇)、定价联动(P6→access F簇)、基药联动(P6→access B簇)、DRG联动(P6→access D簇)、挂网联动(P6→access C簇) |
| ③ 非集采边界 | 集采不主责 | **外置/指路** | 纯医保谈判(→access A簇)、纯挂网准入(→access C簇)、纯注册申报(→drug-reg)、纯市场推广(→drug-market)、纯销售执行(→drug-sales) |

**关键切分说明**：
- **集采 vs 准入**：集采做"集采本身的预判/投标/执行/应对"，准入做"集采对准入路径的影响"（集采后医保支付标准/挂网/进院怎么变）。集采P6给结论，access做深度。
- **集采 vs 注册**：集采提供中标/价格/报量信息，注册提供批件/一致性评价状态支持（→drug-reg）。一致性评价是化药集采门槛，但评价本身归注册。
- **集采 vs 市场**：集采做"集采规则与投标策略"，市场做"集采对市场竞争格局与品牌策略的影响"（→drug-market）。
- **集采 vs 销售**：集采做"中选后供应保障策略"，销售做"终端覆盖/纯销/POA执行"（→drug-sales）。
- **挂网**：集采侧只给"中选价挂网/未中选梯度降价"的结论，挂网准入深度分析归 access C簇。

## 11. 约束与免责

- **结构完整性**：报告须完整保留全部章节与子节；任一维度查不到内容时保留空章节并标「待核实（💭）」，禁止删除章节、合并章节或以「无/暂略」跳过。
- **禁止编造**：中标价/降幅/报量/履约率/集采状态，来源不明标「待核实」。
- **纵向对比（每次必做）**：每次讨论集采必须输出本次vs上次的纵向对比表（§3.2），即使规则没变也要输出并标注"无变化"。不只说变了什么，还要说对企业影响和企业要做什么。
- **横向对比（每次必做）**：每次讨论集采必须输出跨层级/跨联盟/跨省的横向对比表（§3.4），即使品种只涉及一个层级也要做假设对比。对比后必须给结论（哪里最优/最严→企业优先关注/规避）。
- **对药企指导意见（每次必出）**：每次集采分析必须输出对药企的指导意见（§7.2）——当前阶段行动建议+投/不投决策+风险提示+机会识别。
- **药品集采生命周期指导意见（每次必出）**：每次集采分析必须输出该药品未来集采生命周期的指导意见（§7.3）——全路径推演+关键时间节点+各阶段策略+长期演变预判。
- **每次必联网（焊死）**：每次调用本 Skill，第一步必须 WebSearch 获取最新集采信息（§2.5）。禁止凭记忆、禁止用旧数据、禁止跳过联网。**即使"上次查过"，也必须重新联网核实。**
- **来源必须标注URL+日期（焊死）**：每条集采核心数据必须标注来源 URL + 发布日期 + 访问日期（§3.1 格式）。缺 URL 或日期不得作为定论使用。
- **核心数据交叉验证（焊死）**：所有集采核心数据（中标价/降幅/报量/集采状态/规则版本）必须至少 2 个独立来源交叉验证，且至少 1 个来源必须是 P0 官方一手（联采办/医保局/招采平台）。
- **地域差异**：各联盟/各省规则不同时必须按联盟/省分别给出建议，不给"全国通用"笼统结论。
- **品种类型全覆盖**：每个维度必须按品种类型（化药/中成药/中药饮片/生物药）给差异化内容。
- **落盘报告**：须沿用 §7 骨架、校准标注与免责声明，文件名含日期；未要求落盘时仅在对话内输出。
- **免责**：本报告仅供学习/战略参考，不构成投资建议；集采信息以官方最新发布为准；投标策略需结合企业实际情况判断。
